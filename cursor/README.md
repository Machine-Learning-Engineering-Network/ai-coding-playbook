# Cursor Guide

A practical guide to using Cursor as your main AI development environment.

---

## What makes Cursor different

Cursor is not just AI chat — it works directly on your codebase.

It can:

- understand your project

- edit multiple files

- refactor systems

- run iterative improvements

Cursor is built to improve developer productivity by working directly inside the code environment.

---

## Core workflow

1. Open project

2. Highlight or reference code

3. Give structured instructions

4. Review changes

5. Iterate

---

## Best practices

- Be specific with prompts

- Reference exact files when possible

- Keep changes scoped

- Review every change

> Giving precise context improves output quality significantly.

---

## Prompting tips

Bad:

> fix this file

Better:

> Fix this bug without changing existing structure or imports

---

## Advanced usage

### Plan before coding

Ask:

> What files will change? What is the plan?

This improves accuracy and reduces mistakes.

---

### Use context

- reference files directly

- include project rules

- define constraints

---

### Use test-driven prompts

Example:

> Write tests first, then implement and ensure tests pass

This improves reliability and reduces errors.

---

## Limitations

- Can introduce unnecessary changes

- May over-edit files

- Needs human validation

---

## Best practice

> Cursor is best for editing and improving real code, not blind generation.
