# Claude Code Guide

A guide to using Claude for reasoning, debugging, and system design.

---

## What Claude is best at

Claude excels at:
- explaining code
- debugging issues
- writing clean logic
- planning architecture

---

## When to use Claude

Use Claude when:
- something is unclear
- debugging complex issues
- designing systems before coding

---

## Workflow

1. Explain the problem
2. Ask for breakdown
3. Ask for solution
4. refine with follow-ups

---

## Prompting

Bad:
> fix this code

Better:
> Explain the issue, then fix it without changing overall structure

---

## Advanced usage

### Ask for reasoning

Example:
> Explain why this approach works before writing code

---

### Use step-by-step output

Example:
> Break this into steps before implementing

---

## Role in workflow

- Claude → thinking
- Codex → building
- Cursor → editing

---

## Limitation

- Less autonomous than Codex
- Needs structured prompts

---

## Best practice

> Use Claude to think first, then build with other tools.
